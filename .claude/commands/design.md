# /design - UI/UX Design Tasks

Launch the bmad-ux-designer agent for user experience and interface design tasks.

## Usage

```
/design <task description>
```

## When to Use

Use `/design` for UI/UX related tasks:
- Create wireframes and layouts
- Design component systems
- Plan user flows
- Ensure accessibility compliance
- Create responsive designs

## Examples

### Create wireframes
```
/design Create wireframes for the agent search and filter page
```

### Design components
```
/design Design a card component for agent listings with thumbnail, rating, and price
```

### User flow
```
/design Map out the user journey for purchasing an agent
```

### Accessibility audit
```
/design Audit the checkout page for WCAG 2.1 AA compliance
```

### Responsive design
```
/design Create mobile-first responsive layout for the dashboard
```

## What bmad-ux-designer Provides

- **Research**: User personas, journey maps, pain points
- **Information Architecture**: Site structure, navigation
- **Wireframes**: Low and high-fidelity layouts
- **Design System**: Colors, typography, spacing, components
- **Accessibility**: WCAG 2.1 AA compliance
- **Responsive Design**: Mobile, tablet, desktop
- **Component Specs**: Detailed implementation guides

## Design Tools & Standards

- **Framework**: Tailwind CSS + Radix UI
- **Accessibility**: WCAG 2.1 AA
- **Components**: Shadcn/ui compatible
- **Dark mode**: Built-in support
- **Animations**: Framer Motion

## Deliverables

Typically includes:
- Wireframes (ASCII art or description)
- Component specifications (with code examples)
- Design system tokens
- Accessibility guidelines
- User flow diagrams
- Implementation notes for developers
